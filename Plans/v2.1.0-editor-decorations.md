# Live Test Runner — v2.2.0 Editor Inline Decorations

**Branch:** `feat/editor-decorations`  
**Touches:** `ResultStore.ts`, `extension.ts`, new `EditorDecorationManager.ts`, new `CodeLensProvider.ts`, `package.json`  
**Does NOT touch:** `TestExplorerProvider.ts`, `TestResultsProvider.ts`, `testListLayout.js`, `explorer.html`, `results.html`, `styles.css`, `SelectionState.ts`, anything in `packages/runner/`

---

## What v2.2.0 Adds

- **Gutter status icons** — pass ✓ / fail ✗ / running ⟳ / pending ○ next to each `it()`/`test()` line
- **Inline duration text** — muted duration label rendered after the closing paren of each test block
- **CodeLens run button** — `▶ Run` above each `describe`, `it`, and `test` block, visible only inside an active session
- **CodeLens debug button** — `▷ Debug` above each block, launches Jest in debug mode via `vscode.debug.startDebugging` with `--runInBand`
- **Jump to results icon** — `◈ Results` CodeLens that focuses the Test Results panel and selects the matching row

- `◈ Results` only appears on `it`/`test` lines (not `describe`), because results are scoped per test

## Step 1 — `ResultStore.ts`: Add `LineMap`

### 1.1 Add `LineEntry` type

```ts
export type LineEntry = {
  testId:   string
  suiteId:  string
  fileId:   string
  status:   'passed' | 'failed' | 'running' | 'pending'
  duration: number | null
  // runtimeValues will be added here when instrumentor is implemented
  // runtimeValues?: string[]
}
```

### 1.2 Add `LineMap` to the store

```ts
// Inside ResultStore class:
private _lineMap: Map<string, Map<number, LineEntry>> = new Map()
// key: absolute filePath → Map<lineNumber, LineEntry>
```

### 1.3 Add `LineMap` methods

```ts
setLineEntry(filePath: string, line: number, entry: LineEntry): void {
  if (!this._lineMap.has(filePath)) {
    this._lineMap.set(filePath, new Map())
  }
  this._lineMap.get(filePath)!.set(line, entry)
}

getLineMap(filePath: string): Map<number, LineEntry> {
  return this._lineMap.get(filePath) ?? new Map()
}

clearLineMap(filePath: string): void {
  this._lineMap.delete(filePath)
}

clearAllLineMaps(): void {
  this._lineMap.clear()
}
```

### 1.4 Populate `LineMap` in `applyFileResultToStore()`

In `extension.ts`, inside `applyFileResultToStore()`, after writing test results into the store,
iterate over the Jest file result's test cases and call `setLineEntry()` for each one that has
a `location` field:

```ts
// Inside applyFileResultToStore(), after storing test results:
resultStore.clearLineMap(filePath)

for (const suite of jestFileResult.testResults ?? jestFileResult.assertionResults ?? []) {
  for (const test of suite.assertionResults ?? []) {
    if (test.location?.line == null) continue
    resultStore.setLineEntry(filePath, test.location.line, {
      testId:   makeTestId(test.fullName),
      suiteId:  makeSuiteId(suite.ancestorTitles),
      fileId:   fileId,
      status:   mapStatus(test.status),   // 'passed' | 'failed' | 'pending'
      duration: test.duration ?? null,
    })
  }
}
```

On `run-started`, call `resultStore.clearAllLineMaps()` to reset before a fresh full run.

For incremental reruns (single file), `clearLineMap(filePath)` is called before re-populating,
so stale entries for that file are replaced cleanly.

## Step 2 — `EditorDecorationManager.ts` (new file)

This class owns all `TextEditorDecorationType` instances and applies them to open editors.
It is created once in `extension.ts` and disposed on `Stop Testing`.

### 2.1 Decoration types

Create one `TextEditorDecorationType` per gutter state. Icons are SVG files in `resources/icons/`.

```ts
import * as vscode from 'vscode'
import { ResultStore } from './ResultStore'

const DURATION_THRESHOLDS = {
  green:  100,   // < 100ms
  amber:  500,   // 100ms – 500ms
  // > 500ms = red
}

export class EditorDecorationManager {
  private _types = {
    passed:  vscode.window.createTextEditorDecorationType({
      gutterIconPath:  this._icon('passed'),
      gutterIconSize:  'contain',
    }),
    failed:  vscode.window.createTextEditorDecorationType({
      gutterIconPath:  this._icon('failed'),
      gutterIconSize:  'contain',
    }),
    running: vscode.window.createTextEditorDecorationType({
      gutterIconPath:  this._icon('running'),
      gutterIconSize:  'contain',
    }),
    pending: vscode.window.createTextEditorDecorationType({
      gutterIconPath:  this._icon('pending'),
      gutterIconSize:  'contain',
    }),
  }

  constructor(
    private readonly _store: ResultStore,
    private readonly _context: vscode.ExtensionContext,
  ) {}
```

### 2.2 Icon resolution helper

```ts
  private _icon(name: string): vscode.Uri {
    return vscode.Uri.joinPath(this._context.extensionUri, 'resources', 'icons', `${name}.svg`)
  }
```

### 2.3 `applyToEditor(editor)`

Called whenever an editor becomes active or a file result arrives for the open file.

```ts
  applyToEditor(editor: vscode.TextEditor): void {
    const filePath = editor.document.uri.fsPath
    const lineMap  = this._store.getLineMap(filePath)

    const buckets: Record<string, vscode.DecorationOptions[]> = {
      passed: [], failed: [], running: [], pending: [],
    }

    for (const [line, entry] of lineMap) {
      const range = new vscode.Range(line - 1, 0, line - 1, 0)

      const durationText = entry.duration != null
        ? `  ${entry.duration}ms`
        : ''

      const durationColor = entry.duration == null ? ''
        : entry.duration < DURATION_THRESHOLDS.green ? 'var(--vscode-terminal-ansiGreen)'
        : entry.duration < DURATION_THRESHOLDS.amber ? 'var(--vscode-terminal-ansiYellow)'
        : 'var(--vscode-terminal-ansiRed)'

      const decoration: vscode.DecorationOptions = {
        range,
        renderOptions: {
          after: {
            contentText:     durationText,
            color:           durationColor,
            fontStyle:       'normal',
            margin:          '0 0 0 16px',
          },
        },
      }

      buckets[entry.status]?.push(decoration)
    }

    // Apply all four types — clear any that have no entries
    for (const [state, opts] of Object.entries(buckets)) {
      editor.setDecorations(this._types[state as keyof typeof this._types], opts)
    }
  }
```

### 2.4 `clearEditor(editor)` and `dispose()`

```ts
  clearEditor(editor: vscode.TextEditor): void {
    for (const type of Object.values(this._types)) {
      editor.setDecorations(type, [])
    }
  }

  clearAll(): void {
    for (const editor of vscode.window.visibleTextEditors) {
      this.clearEditor(editor)
    }
  }

  dispose(): void {
    this.clearAll()
    for (const type of Object.values(this._types)) {
      type.dispose()
    }
  }
}
```

## Step 4 — `CodeLensProvider.ts` (new file)

Provides `▶ Run`, `▷ Debug`, and `◈ Results` lenses above each block.

### 4.1 Detection strategy

No AST parsing. Use a line-by-line regex scan at `provideCodeLenses()` time. This runs fast
on every file open and is re-triggered by `onDidChangeCodeLenses`.

```ts
const BLOCK_PATTERN = /^\s*(describe|it|test)\s*[\.(]/
```

For each matching line, emit lenses. The `resolve` step (which line maps to which test) uses
the `LineMap` — if there is no entry for that line yet (no result), `▶ Run` still appears but
`◈ Results` is omitted.

### 4.2 Full implementation

```ts
import * as vscode from 'vscode'
import { ResultStore } from './ResultStore'

const BLOCK_PATTERN  = /^\s*(describe|it|test)\s*[\.(]/
const SUITE_PATTERN  = /^\s*describe\s*[\.(]/

export class LiveTestCodeLensProvider implements vscode.CodeLensProvider {
  private _onDidChange = new vscode.EventEmitter<void>()
  readonly onDidChangeCodeLenses = this._onDidChange.event

  constructor(private readonly _store: ResultStore) {}

  refresh(): void {
    this._onDidChange.fire()
  }

  provideCodeLenses(document: vscode.TextDocument): vscode.CodeLens[] {
    const lenses: vscode.CodeLens[] = []
    const filePath = document.uri.fsPath
    const lineMap  = this._store.getLineMap(filePath)

    for (let i = 0; i < document.lineCount; i++) {
      const text = document.lineAt(i).text
      if (!BLOCK_PATTERN.test(text)) continue

      const lineNumber = i + 1   // 1-based, matches Jest location.line
      const isSuite    = SUITE_PATTERN.test(text)
      const entry      = lineMap.get(lineNumber)
      const range      = new vscode.Range(i, 0, i, 0)

      // ▶ Run
      lenses.push(new vscode.CodeLens(range, {
        title:     '▶ Run',
        command:   'liveTestRunner.rerunFromEditor',
        arguments: [filePath, lineNumber],
      }))

      // ▷ Debug
      lenses.push(new vscode.CodeLens(range, {
        title:     '▷ Debug',
        command:   'liveTestRunner.debugFromEditor',
        arguments: [filePath, lineNumber],
      }))

      // ◈ Results — only on it/test lines with a known result
      if (!isSuite && entry) {
        lenses.push(new vscode.CodeLens(range, {
          title:     '◈ Results',
          command:   'liveTestRunner.focusResult',
          arguments: [entry.fileId, entry.suiteId, entry.testId],
        }))
      }
    }

    return lenses
  }
}
```

### 4.3 Session guard

The `CodeLensProvider` is only registered while a session is active.

In `extension.ts`:

```ts
let codeLensDisposable: vscode.Disposable | undefined

function startSession() {
  // ... existing start logic ...
  codeLensDisposable = vscode.languages.registerCodeLensProvider(
    { scheme: 'file', pattern: getTestFileGlob() },
    codeLensProvider,
  )
  context.subscriptions.push(codeLensDisposable)
}

function stopSession() {
  // ... existing stop logic ...
  codeLensDisposable?.dispose()
  codeLensDisposable = undefined
  decorationManager.clearAll()
}
```

`getTestFileGlob()` reads `liveTestRunner.testFilePatterns` from settings and converts to a
VS Code `GlobPattern`.

### 5.4 Register new commands

```ts
// Reruns the test or suite at a given line
vscode.commands.registerCommand('liveTestRunner.rerunFromEditor',
  (filePath: string, line: number) => {
    const entry = resultStore.getLineMap(filePath).get(line)
    if (!entry) {
      // No result yet — run the whole file
      runFiles([filePath])
      return
    }
    rerunScope({ fileId: entry.fileId, suiteId: entry.suiteId, testId: entry.testId })
  }
)

// Launches Jest in debug mode for the test at a given line
vscode.commands.registerCommand('liveTestRunner.debugFromEditor',
  async (filePath: string, line: number) => {
    const entry  = resultStore.getLineMap(filePath).get(line)
    const binary = await binaryResolver.resolve(projectRoot)
    const folder = vscode.workspace.getWorkspaceFolder(vscode.Uri.file(filePath))

    // If no LineMap entry yet (test never run), debug the whole file.
    // If entry exists, scope to exactly that test via --testNamePattern.
    const args: string[] = [filePath]
    if (entry) {
      args.push('--testNamePattern', escapeRegex(entry.testId))
    }
    args.push(
      '--runInBand',    // required — debugger attaches to a single process
      '--no-coverage',  // skip coverage instrumentation when debugging
    )

    await vscode.debug.startDebugging(folder, {
      type:      'node',
      request:   'launch',
      name:      'Debug Jest test',
      program:   binary,
      args,
      cwd:       projectRoot,
      console:   'integratedTerminal',
      skipFiles: ['<node_internals>/**'],
    })
  }
)

// Focuses the Test Results panel and selects the matching row
vscode.commands.registerCommand('liveTestRunner.focusResult',
  (fileId: string, suiteId: string, testId: string) => {
    // Reveal the Test Results panel
    vscode.commands.executeCommand('liveTestRunner.testResults.focus')
    // Broadcast scope selection — webview will scroll to and highlight the row
    selectionState.select({ level: 'test', fileId, suiteId, testId })
    testResultsProvider.postMessage({
      type: 'scope-changed',
      scope: 'test', fileId, suiteId, testId,
    })
  }
)
```

### `escapeRegex` helper

`--testNamePattern` accepts a regex. Test names can contain characters like `(`, `)`, `.` that
need escaping to be treated as literals:

```ts
function escapeRegex(str: string): string {
  return str.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')
}
```

## Behaviour Summary

| Trigger | What happens |
|---------|-------------|
| Session starts | `CodeLensProvider` registered, all open editors get decorations applied |
| File result arrives | `LineMap` rebuilt for that file, decorations and lenses refresh |
| Test is running | Gutter shows `⟳` amber spinner for that line |
| Test passes | Gutter shows `✓` green, duration in green/amber/red after the line |
| Test fails | Gutter shows `✗` red, duration shown |
| No result yet | Gutter shows `○` grey, no duration, `◈ Results` not shown |
| `▶ Run` clicked | Reruns that test if in `LineMap`, else runs whole file |
| `▷ Debug` clicked | Launches Jest via `vscode.debug.startDebugging` with `--runInBand --no-coverage`, scoped to that test via `--testNamePattern` |
| `◈ Results` clicked | Focuses Test Results panel, scrolls to and selects that test row |
| Session stops | All decorations cleared, lenses removed, `LineMap` cleared |
| File closed/opened | Decorations reapplied from existing `LineMap` on `onDidChangeActiveTextEditor` |
