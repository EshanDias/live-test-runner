# v2.0.0 Implementation Progress

**Version decision:** v2.0.0 — replacing native VS Code Testing API with fully custom webview UI.  
**Plan source:** Plans/v1.1.0.md  
**Started:** 2026-04-03

---

## Why v2.0.0?
The plan replaces the entire `vscode.TestController` / `TestRun` API with custom `WebviewViewProvider` panels. This is a breaking architectural change — the native Test Results panel is gone, replaced by our own sidebar + panel views.

---

## Branch Strategy
Each feature area gets its own branch, merged to `develop`, then `main` on release.

## If Context Runs Out — Resume From Here

**Last completed step:** Phase 1 fully implemented and merged to `develop`. Both custom webview providers live, compile cleanly, extension.ts wired up.

**Next step (Phase 2):** `feat/editor-decorations` — gutter icons + inline duration using `vscode.window.createTextEditorDecorationType`. Map Jest JSON `location.line` to source positions.

**Key decisions already made (see Plans/v1.1.0.md §Decisions Made):**
- Two separate webviews sharing one testListLayout.js
- Scoping logic lives in the extension, not the webview
- SelectionState broadcasts `scope-changed` to both webviews
- Auto-expand failed files on run complete
- Log auto-scroll locks if user has scrolled up
- Duration thresholds: test < 100ms green / 100-500ms amber / >500ms red

**Message protocol is fully documented in Plans/v1.1.0.md §Architecture.**
